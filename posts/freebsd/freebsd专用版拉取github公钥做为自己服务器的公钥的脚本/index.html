Freebsd专用版拉取github公钥做为自己服务器的公钥的脚本


  • 脚本只能是root用户运行～

  • 脚本会自动修改sshd_config的配置，省心省力，基本上就两步，导入，查看一下，完事！

  • github如何上传公钥，请自行查询，一劳永逸的事,本身就有，就不用再上传了，通用的。

  • 下面执行一键命令（FreeBSD）

curl -fsSL https://raw.githubusercontent.com/hanigege/scripts/main/freebsd_ssh_key_manage.sh \
-o freebsd_ssh_key_manage.sh \
&& chmod +x freebsd_ssh_key_manage.sh \
&& sh freebsd_ssh_key_manage.sh

配置项核对

  • UsePAM no: 正确。在 FreeBSD 中，关闭 PAM 是防止系统通过其他交互方式询问密码的最彻底手段。
  • PermitRootLogin prohibit-password: 正确。这允许 Root 用户通过密钥登录，但绝对禁止使用密码。
  • PasswordAuthentication no: 核心正确。直接关闭了标准的密码认证通道。
  • PubkeyAuthentication yes: 核心正确。这是开启密钥登录的开关。
  • ChallengeResponseAuthentication no: 正确。关闭挑战响应认证。
  • KbdInteractiveAuthentication no: 非常正确。这是 FreeBSD 环境下的关键，关闭了键盘交互式认证，彻底堵死密码入口。
