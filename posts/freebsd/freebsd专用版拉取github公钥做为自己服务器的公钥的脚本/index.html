Freebsd专用版拉取github公钥做为自己服务器的公钥的脚本


  • 脚本会自动修改sshd_config的配置，省心省力，基本上就两步，导入，查看一下，完事！

  • github如何上传公钥，请自行查询，一劳永逸的事,本身就有，就不用再上传了，通用的。

  • 下面执行一键命令（FreeBSD）

  • 仅限root用户登录使用，配置的是root用户的证书，请在root下执行脚本

1. 仅限root用户登录使用，配置的是root用户的证书

fetch -o freebsd_ssh_key_manage.sh https://raw.githubusercontent.com/hanigege/scripts/main/freebsd_ssh_key_manage.sh && chmod +x freebsd_ssh_key_manage.sh && ./freebsd_ssh_key_manage.sh

root用户配置项核对

  • UsePAM no: 正确。在 FreeBSD 中，关闭 PAM 是防止系统通过其他交互方式询问密码的最彻底手段。
  • PermitRootLogin prohibit-password: 正确。这允许 Root 用户通过密钥登录，但绝对禁止使用密码。
  • PasswordAuthentication no: 核心正确。直接关闭了标准的密码认证通道。
  • PubkeyAuthentication yes: 核心正确。这是开启密钥登录的开关。
  • ChallengeResponseAuthentication no: 正确。关闭挑战响应认证。
  • KbdInteractiveAuthentication no: 非常正确。这是 FreeBSD 环境下的关键，关闭了键盘交互式认证，彻底堵死密码入口。

2. 仅限普通用户登录使用，配置的是普通用户的证书，请在root下执行脚本

fetch -o ssh_manager.sh https://raw.githubusercontent.com/hanigege/scripts/main/freebsd_user_ssh_manager.sh && chmod +x ssh_manager.sh && ./ssh_manager.sh

普通用户配置项核对

UsePAM no
PermitRootLogin no
PasswordAuthentication no
PubkeyAuthentication yes
ChallengeResponseAuthentication no
KbdInteractiveAuthentication no

这个结果正是你所需要的终极安全状态：

  • PermitRootLogin no：彻底掐断了 Root 直接登录。

  • PasswordAuthentication no 及相关配置：完全关闭了密码验证通道。

  • PubkeyAuthentication yes：仅放行持有对应私钥的用户（即你刚刚配置的那个普通用户）。

⚠️ 强烈建议（防翻车操作）： 千万别急着关掉当前的 Root 终端窗口！ 请保持当前这个连接不断开，立刻新建一个终端窗口，用你的普通用户加上刚才生成的私钥（或导入的对应私钥）尝试登录一下这台服务器。
