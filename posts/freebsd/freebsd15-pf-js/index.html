FreeBSD 15 PF 终极交互式管理脚本


开启防火墙的步骤

第一步：启用 PF 模块

执行以下命令，确保重启后自动开启：

sysrc pf_enable="YES"
sysrc pflog_enable="YES"

第二步：创建一个基础配置文件 （可选，一般都有）

如果 /etc/pf.conf 不存在，pf 将无法启动。先创建一个最简单的配置（只放行 SSH，假设你的网卡是 vtnet0，请根据 ifconfig 实际名称修改）：

PF

# /etc/pf.conf
ext_if = "vtnet0"
set skip on lo0
scrub in all

# 默认拒绝所有入站，允许所有出站
block in all
pass out all keep state

# 放行 SSH (22 端口)，防止断开连接 
pass in on $ext_if proto tcp from any to any port 22 keep state

第三步：手动加载并启动

# 加载内核模块
kldload pf
# 启动服务
service pf start

一键执行脚本

curl -fsSL https://raw.githubusercontent.com/hanigege/scripts/main/pf_manager.sh -o pf_manager.sh && sh pf_manager.sh

成功的步骤方法（写进教程的最后一步）：

为了防止服务器突然断电或意外重启导致还未查看的“内存黑名单”丢失，强烈建议在教程末尾补充一句：设置定时任务自动落盘。

执行以下命令即可：

echo "0 * * * * root /sbin/pfctl -t blacklist -T show > /etc/pf_blacklist.txt 2>/dev/null" >> /etc/crontab

这样，你的 FreeBSD 系统就拥有了一套完整的自防御、自恢复、抗重启的防火墙体系。
