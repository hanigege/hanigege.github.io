Freebsd普通用户如何使用sudo

我个人不建议安装开启sudo,因为普通用户用自己的密码就可以提升管理权限。个人理解还是完全隔离的好

**下面只提供知识点参考**～

安装freebsd时已经添加好普通用户的，可以直接从下面可以用 visudo 打开 sudo 配置文件 这里直接修改配置文件就可以了。

你看到的提示：

gary is not in the sudoers file.
This incident has been reported to the administrator.

意思是：当前用户 gary 没有权限使用 sudo，因此不能安装软件或执行需要管理员权限的操作。

解决办法（需要 root 用户操作） 你需要切换到 root 用户，然后把 gary 添加到 sudo 权限组。步骤如下：

✅ 1. 用 root 用户登录，或者用 su 切换到 root：

su -

系统会提示你输入 root 的密码，不是 gary 的密码。

✅ 2. 安装 sudo（如果还没装）：

pkg install sudo

✅ 3. 把 gary 用户添加到 wheel 组：(如果安装系统时已加入wheel组就不需要下面的再次操作）

pw usermod gary -G wheel

默认 FreeBSD 的 sudoers 文件允许 wheel 组的用户使用 sudo。

✅ 4. 确认 sudoers 配置中允许 wheel 组使用 sudo：

可以用 visudo 打开 sudo 配置文件：（默认vi命令打开）

visudo

指定vim命令打开visudo文件：

EDITOR=vim visudo

确保有以下内容（没有注释 #）：

%wheel ALL=(ALL:ALL) ALL

特别说明：打开visudo文件后，会看到下面三种，只需要去掉带%wheel ALL=(ALL:ALL) ALL前面的#号就行了：

root ALL=(ALL:ALL) ALL

## Uncomment to allow members of group wheel to execute any command
# %wheel ALL=(ALL:ALL) ALL

## Same thing without a password
# %wheel ALL=(ALL:ALL) NOPASSWD: ALL

## Uncomment to allow members of group sudo to execute any command
# %sudo ALL=(ALL:ALL) ALL

✅ 5. 现在用 gary 登录，就能用 sudo 了：

sudo pkg install bash .....

普通用户添加免密权限

在文件末尾添加以下内容（将 felix 替换为你实际的用户名）：

felix ALL=(ALL) NOPASSWD: ALL

如何解决“普通用户密码泄露导致系统沦陷”的问题？

你的担忧是真实的，但解决方式是加固 sudo 配置，而不是退回 su -。以下是业界标准的成功应对方法与步骤：

方法一：细化命令白名单（最佳实践）

不要给普通用户 ALL=(ALL:ALL) ALL 权限。通过 visudo 严格限制其能执行的命令。


# 只允许 user1 以 root 身份重启 nginx，即使其密码被盗，攻击者也只能重启 nginx
user1 ALL=(root) /bin/systemctl restart nginx

方法二：配置 sudo 要求验证 root 密码

如果你既想要 sudo 的审计功能，又想要 su - 的密码隔离安全性，可以修改 sudo 行为，让其验证目标用户（root）的密码，而不是执行者本人的密码。


# 执行 visudo，在 Defaults 区域添加以下配置：
Defaults rootpw

方法三：引入多因素认证 (MFA / 2FA)

即使普通用户密码被暴力破解或泄露，没有动态验证码，攻击者依然无法执行 sudo。


# Debian/Ubuntu 示例，配置 PAM 引入 Google Authenticator
sudo apt install libpam-google-authenticator
# 在 /etc/pam.d/sudo 文件末尾添加：
auth required pam_google_authenticator.so

亲，以下是实现权限细分和双因素认证的完整配置步骤。

一、 细化命令白名单 (sudoers 配置)

结论： 通过 visudo 定义命令别名（Cmnd_Alias），将特定绝对路径的命令授权给指定用户，实现最小权限原则。

操作步骤：

切换到 root 或使用具备全部特权的账号，执行 visudo 编辑配置文件。

在文件末尾添加以下配置（以允许普通用户 gary 重启 Nginx 和查看日志为例）：


# 1. 定义命令别名 (必须使用绝对路径，可包含多条命令，逗号分隔)
Cmnd_Alias NGINX_CMDS = /bin/systemctl restart nginx, /bin/systemctl reload nginx
Cmnd_Alias LOG_CMDS = /usr/bin/tail -f /var/log/nginx/access.log

# 2. 授权普通用户执行上述命令
# 选项A：执行命令时需要验证普通用户自己的密码
gary ALL=(root) NGINX_CMDS, LOG_CMDS

# 选项B：执行命令时免密码验证 (适用于低风险、高频操作)
# gary ALL=(root) NOPASSWD: NGINX_CMDS, LOG_CMDS

二、 配置 sudo 结合双因素认证 (2FA)

结论： 利用 PAM (Pluggable Authentication Modules) 结合 Google Authenticator，在执行 sudo 时强制要求验证手机上的动态密码。即使系统密码泄露，攻击者也无法提权。

  1. Debian/Ubuntu 环境配置步骤

1. 安装 Google Authenticator PAM 模块

sudo apt update
sudo apt install libpam-google-authenticator -y

2. 切换到需要配置 2FA 的普通用户 (如 gary)，初始化密钥

su - gary
google-authenticator

按照终端提示一路输入 ‘y’。使用手机上的 Authenticator App 扫描终端输出的二维码。

3. 编辑 sudo 的 PAM 配置文件

sudo nano /etc/pam.d/sudo

4. 在文件中 @include common-auth 的下方，添加以下行：

auth required pam_google_authenticator.so

5. 保存文件。打开一个【全新】的终端窗口测试 sudo 命令，此时会提示 Verification code:

  1. FreeBSD 14.2 环境配置步骤

Bash

1. 安装模块

sudo pkg install pam_google_authenticator

2. 切换到普通用户并生成密钥

su - gary
google-authenticator

扫码绑定，并妥善保存 emergency scratch codes

3. 编辑 sudo 的 PAM 配置文件

注意：FreeBSD 下 sudo 的 PAM 配置通常位于 /usr/local/etc/pam.d/sudo

sudo vi /usr/local/etc/pam.d/sudo

4. 在文件的 auth 规则部分添加以下行 (注意使用绝对路径)：

auth required /usr/local/lib/pam_google_authenticator.so

5. 保存文件。打开新终端进行测试。

防翻车警告：

在修改 PAM 配置文件（/etc/pam.d/sudo 或 /usr/local/etc/pam.d/sudo）时，绝对不要关闭当前拥有 root 权限的终端窗口。务必另开一个普通用户的终端窗口测试 sudo -i 或 sudo ls 能否正常通过密码+2FA验证。如果配置出错导致 sudo 瘫痪，你可以在未关闭的 root 窗口中直接撤销刚才的修改。

亲，结论是：需要开启 %wheel ALL=(ALL:ALL) ALL。

并且，强烈建议你结合 wheel 组来限制权限，而不是向系统内所有用户开放。如果开启了你配置文件中的 ALL ALL=(ALL:ALL) ALL，意味着系统里的任何普通用户（包括那些因为某些服务漏洞被黑客临时获取的低权限账户）都能去尝试输入 root 密码进行提权。这扩大了攻击面。

最安全的做法是：只有被加入 wheel 组的核心管理员，才有资格使用 sudo，并且在执行时必须验证 root 的密码。

成功配置步骤

使用 visudo 编辑，找到并修改以下几处（去掉前面的 # 号）：

  1. 开启验证目标用户（root）密码：

Defaults targetpw  # Ask for the password of the target user
  1. 开启对应密码提示（强烈建议，终端会明确提示你正在输入 root 密码，防止混淆）：

Defaults passprompt="%p's password:"
  1. 授权 wheel 组使用 sudo：

%wheel ALL=(ALL:ALL) ALL
  1. 保持这行被注释（绝对不要开启）：

# ALL ALL=(ALL:ALL) ALL  # WARNING: only use this together with 'Defaults targetpw'

收尾工作

保存退出后，你需要确保你的普通用户已经在 wheel 组内。

FreeBSD 下添加用户到 wheel 组： pw groupmod wheel -m 你的用户名

Linux 下添加用户到 wheel 组： usermod -aG wheel 你的用户名

新开一个终端窗口测试执行 sudo ls，此时提示语会变成 root’s password:，输入 root 密码即可通过验证。
