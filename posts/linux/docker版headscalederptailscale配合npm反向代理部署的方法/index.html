Docker版headscale&derp&tailscale配合npm反向代理回家的方法

说明

因为太实用了，不管是回家，还是建立私有隧道，或者是添加节点等等吧都行,不过也有硬性要求，就是家里要使用pve，软路由是ros（其它的路由系统道理也是相通的）才可以部署！

开始安装

docker一键安装

curl -fsSL https://raw.githubusercontent.com/hanigege/scripts/main/docker.sh -o docker.sh && sh docker.sh

docker.sh 该脚本用于自动安装 docker 以及 docker 插件(composes、buildx 等), 同时该脚本将会自动配置 /etc/docker/daemon.json 文件, 目前针对 docker daemon 的配置逻辑如下: 默认开启 init 支持, 防止容器产生僵尸进程 切换 docker 存储目录为 /data/docker, 此后所有 docker 数据将会存放于此 限制容器日志文件大小, 单个容器最多 5 个日志文件, 单个日志文件最大 10m, 防止容器 stdout 过量输出导致占满磁盘空间 开启 live-restore 防止 docker daemon 意外重启导致容器重启 Debian、Redhat 系列系统默认 CGroups Driver 切换为 systemd, Alpine 系统保持默认 cgroupfs v2 该脚本安装完成后默认会 Pin 住 docker 相关软件包, 防止自动升级导致容器重启或出现不可逆故障.

1. Tailscale容器安装

  • 建立目录
mkdir -p ~/data/docker_data/tailscale
cd ~/data/docker_data/tailscale
  • 建立docker-compose.yaml
nano docker-compose.yaml

services:
  tailscale:
    image: tailscale/tailscale:latest
    container_name: tailscale-node
    hostname: home-gateway
    restart: always
    network_mode: "host" # 必须用 host 模式才能作为网关和访问宿主机网段
    cap_add:
      - NET_ADMIN
      - NET_RAW
    devices:
      - /dev/net/tun:/dev/net/tun
    volumes:
      - ./tailscale-state:/var/lib/tailscale
      - /var/run/dbus/system_bus_socket:/var/run/dbus/system_bus_socket
    environment:
      - TS_STATE_DIR=/var/lib/tailscale
      - TS_USERSPACE=false
      # 第一次运行建议手动执行 login 命令，或者在这里填入 Headscale 的 AuthKey
      # - TS_AUTHKEY=你的Key
    # 启动参数：开启子网通告和出口节点功能
    command: tailscaled --port=41641 --tun=userspace-networking --socks5-server=0.0.0.0:1055

2. derper中继容器安装

  • 建立目录
mkdir ~/data/docker_data/derps
cd ~/data/docker_data/derps
  • 建立docker-compose.yaml
nano docker-compose.yaml

services:
  derper:
    image: fredliang/derper
    container_name: derper
    restart: always
    ports:
      - "8082:80"
      - "3478:3478/udp"
    # 去掉 --hostname，由 NPM 统一接管域名
    command: /app/derper --a :80 --http-port 80 --stun --verify-clients=false
    environment:
      - DERP_ADDR=:80
      - DERP_HTTP_PORT=80
  • VERIFY_CLIENTS=false # 暂设为 false 以确保连接成功（意思是不用认证了）

ros篇

都配置好以后，后面需要手动做好ros的端口映射、回流，（本站有这方面的教程） 即：分别做好端口12335对外的转发服务以及derps对外服务的3478端口

做好端口映射，都配置好以后，ros还需要特别配置如下：

ros再配置 （下面的配置里还是以41641端口为例）

动态 IP 自动同步脚本” 和 “强制端口锁定规则” 重新组合 导入，请更改下面配置中你的内网ip 如更改过tailscale的docker-compose.yaml配置里的41641端口，下面的41641端口也要更改成一样的，还有对应的内网ip.

# 1. 【清场】删除所有之前乱掉的 Tailscale 规则和脚本
/ip firewall nat remove [find comment~"Tailscale"]
/ip firewall mangle remove [find comment~"Tailscale"]
/ip firewall raw remove [find comment~"Tailscale"]
/system scheduler remove [find name="Sync_Tailscale_WAN"]
/system script remove [find name="Update_Tailscale_NAT"]

# 2. 【核心规则】出站锁定 (锁定 41641 端口)
# 我们不写死 to-addresses，先填 1.1.1.1 占位，靠脚本去改
/ip firewall nat add chain=srcnat action=src-nat to-addresses=1.1.1.1 to-ports=41641 \
    protocol=udp src-address=10.20.20.8 comment="Tailscale_Outbound_Fixed" place-before=0

# 3. 【核心规则】入站映射
/ip firewall nat add chain=dstnat action=dst-nat protocol=udp dst-port=41641 \
    in-interface=pppoe-out1 to-addresses=10.20.20.8 to-ports=41641 \
    comment="Tailscale_Inbound_Fixed"

# 4. 【组合脚本】获取 pppoe-out1 的实时公网 IP 并填入上面的 Rule 0
/system script add name="Update_Tailscale_NAT" source={
    :local currentIP [/ip address get [find interface=pppoe-out1] address];
    :set currentIP [:pick $currentIP 0 [:find $currentIP "/"]];
    :if ([:len $currentIP] > 0) do={
        /ip firewall nat set [find comment="Tailscale_Outbound_Fixed"] to-addresses=$currentIP;
        # :log info ("Tailscale NAT 已同步为公网 IP: " . $currentIP);
    }
}

# 5. 【自动化】设置每分钟运行一次脚本
/system scheduler add name="Sync_Tailscale_WAN" interval=1m on-event="Update_Tailscale_NAT"

# 6. 【最关键的临门一脚】立即执行脚本并清空所有旧的错误缓存
/system script run Update_Tailscale_NAT
/ip firewall connection remove [find]

导入后，如下图，关注导入规则的位置 代码段 导入后，NAT项会有一个在最上面，一个在最下面，保持这样，不要调整位置，把其它转发的端口放中间，回流的规则放在倒数第二条，下图里我已经把41641改为了7829，这个端口可以改为你想用的端口，注意：同时tailscale的docker-compose.yaml配置里的41641端口也要修改，如下图我都改为了7829，默认是41641，明白了吧～！(重要的地方我说了两遍）

优化（必须操作）在npm中，下面的域名的配置里添加以下：

derps.jgaga.xyz 高级中填写：

location / {
    proxy_pass http://10.20.20.8:8082;
    proxy_http_version 1.1;

    # 禁用缓冲实现即时转发
    proxy_buffering off;
    proxy_request_buffering off;

    # 允许长连接保活
    proxy_socket_keepalive on;
    tcp_nodelay on;

    # --- 核心：WebSocket 支持 ---
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";

    # --- 致命修正：DERP 也必须感知公网 12335 端口 ---
    proxy_set_header Host $host:12335; 
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
}

hs.jgaga.xyz 高级中填写：

location / {
    proxy_pass http://10.20.20.8:8080;

    # --- 1. 极致回收：解决网络切换后的僵死 (关键) ---
    # 将超时缩短到运营商 NAT 回收阈值以下 (建议 15-20s)
    proxy_connect_timeout 5s; 
    proxy_send_timeout 20s;
    proxy_read_timeout 20s; 
    
    # --- 2. 彻底关闭缓存与缓冲 (解决实时性) ---
    proxy_buffering off;
    proxy_request_buffering off;
    proxy_cache off;
    proxy_http_version 1.1;

    # --- 3. 动态头：提高兼容性 ---
    # 使用 $http_host 自动捕获客户端发来的包含端口的完整 Host
    proxy_set_header Host $http_host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;

    # --- 4. WebSocket 支持 ---
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";

    # --- 5. 跨域放行 ---
    add_header 'Access-Control-Allow-Origin' '*' always;
    add_header 'Access-Control-Allow-Methods' 'GET, POST, OPTIONS, PUT, DELETE' always;
    add_header 'Access-Control-Allow-Headers' '*' always;

    if ($request_method = 'OPTIONS') {
        return 204;
    }
}

## 下面是linux下的tailscale客户端做出口网关的命令组合：
linux客户端注册服务端开启各种功能，各命令参数配合使用会有不同的功能。

#### 例如：
如果你确实想重新触发登录流程：
如果你想彻底退出当前账号并重新获取登录链接，可以执行命令：

```bash
docker exec tailscale-node tailscale up \
  --login-server https://hs.jgaga.xyz:12335 \
  --advertise-routes=10.20.20.0/24 \
  --advertise-exit-node \
  --accept-routes \
  --accept-dns=false \
  --force-reauth \
  --reset

–accept-dns=true \ #接受服务端指定的dns

–advertise-exit-node \ # 指定为出口网关

–advertise-routes=10.20.20.0/24 \ #允许其它节点访问内网段

去掉–accept-routes \ #这是让这台设备不让其它设备看到自己的内网– advertise-routes=10.20.20.0/24 \

如果是第三方出口网关linux节点：

第三方节点设备可以用tailscale官方的一键安装命令即可：

curl -fsSL https://tailscale.com/install.sh | sh

既然你只想让它做出口网关（Exit Node）和子网路由器（Subnet Router）， 请使用以下去掉了 –accept-routes 的命令：

docker exec tailscale-node tailscale up \
  --login-server=https://hs.jgaga.xyz:12335 \
  --accept-dns=false \
  --advertise-exit-node \
  --reset
