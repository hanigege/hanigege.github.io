~/data/docker_data/
├── headscale及headscale-UI/ # 服务端与配置
├── derps/              # 中继服务器
└── tailscale/          # 网关节点

开始安装

docker一键安装 (这脚本不是我写的，出自mritd) 本站没像这样注明的，都是我写的，请放心使用。这脚本佬也信的过

curl -fsSL https://raw.githubusercontent.com/hanigege/scripts/main/docker.sh -o docker.sh && sh docker.sh

docker.sh 该脚本用于自动安装 docker 以及 docker 插件(composes、buildx 等), 同时该脚本将会自动配置 /etc/docker/daemon.json 文件, 目前针对 docker daemon 的配置逻辑如下: 默认开启 init 支持, 防止容器产生僵尸进程 切换 docker 存储目录为 /data/docker, 此后所有 docker 数据将会存放于此 限制容器日志文件大小, 单个容器最多 5 个日志文件, 单个日志文件最大 10m, 防止容器 stdout 过量输出导致占满磁盘空间 开启 live-restore 防止 docker daemon 意外重启导致容器重启 Debian、Redhat 系列系统默认 CGroups Driver 切换为 systemd, Alpine 系统保持默认 cgroupfs v2 该脚本安装完成后默认会 Pin 住 docker 相关软件包, 防止自动升级导致容器重启或出现不可逆故障.

建立docker本地目录

mkdir -p ~/data/docker_data

1. Tailscale容器安装

  • 建立目录
mkdir ~/data/docker_data/tailscale
cd ~/data/docker_data/tailscale
  • 建立docker-compose.yaml
nano docker-compose.yaml

services:
  tailscale:
    image: tailscale/tailscale:latest
    container_name: tailscale-node
    hostname: home-gateway
    restart: always
    network_mode: "host" # 必须用 host 模式才能作为网关和访问宿主机网段
    cap_add:
      - NET_ADMIN
      - NET_RAW
    devices:
      - /dev/net/tun:/dev/net/tun
    volumes:
      - ./tailscale-state:/var/lib/tailscale
      - /var/run/dbus/system_bus_socket:/var/run/dbus/system_bus_socket
    environment:
      - TS_STATE_DIR=/var/lib/tailscale
      - TS_USERSPACE=false
      # 第一次运行建议手动执行 login 命令，或者在这里填入 Headscale 的 AuthKey
      # - TS_AUTHKEY=你的Key
    # 启动参数：开启子网通告和出口节点功能
    command: tailscaled --port=7829 --tun=userspace-networking --socks5-server=0.0.0.0:1055

2. derper中继容器安装

  • 建立目录
mkdir ~/data/docker_data/derps
cd ~/data/docker_data/derps
  • 建立docker-compose.yaml
nano docker-compose.yaml

services:
  derper:
    image: fredliang/derper
    container_name: derper
    restart: always
    ports:
      - "8082:80"
      - "3478:3478/udp"
    # 去掉 --hostname，由 NPM 统一接管域名
    command: /app/derper --a :80 --http-port 80 --stun --verify-clients=false
    environment:
      - DERP_ADDR=:80
      - DERP_HTTP_PORT=80

3. headscale容器安装(包含Headscale UI WEB管理容器）

  • 建立目录
mkdir ~/data/docker_data/headscale
cd ~/data/docker_data/headscale
  • 建立docker-compose.yaml
nano docker-compose.yaml

services:
  # 服务 1：Headscale 主程序
  headscale:
    image: headscale/headscale:latest
    container_name: headscale-server
    restart: always
    volumes:
      - ./config:/etc/headscale
      - ./data:/var/lib/headscale
    ports:
      - "8080:8080"
    command: serve

  # 服务 2：Headscale UI 界面 (注意：现在它和上面的 headscale 对齐了)
  headscale-ui:
    image: ghcr.io/gurucomputing/headscale-ui:latest
    container_name: headscale-ui
    restart: always
    # 建议映射一个端口，方便你在 NPM 还没配好时先用内网 IP 验证一下
    ports:
      - "9443:8080"

4. Nginx Proxy Manager容器安装

  • 建立目录
mkdir ~/data/docker_data/npm
cd ~/data/docker_data/npm
  • 建立docker-compose.yaml
nano docker-compose.yaml

services:
  app:
    image: 'docker.io/jc21/nginx-proxy-manager:latest'
    restart: unless-stopped
    ports:
      - '12332:80'
      - '81:81'
      - '12335:443'
    volumes:
      - ./data:/data
      - ./letsencrypt:/etc/letsencrypt
  • 说明：对外的https端口为12335，内网管理端口为81 对外80端口为12332

分别启动容器

docker-compose up -d

  • 注意：为了确保 derper 能够正确验证客户端，derper 应该在 tailscaled 启动之后再运行

  • Nginx Proxy Manager里的域名下图这块，先自行建立好：不会可以看咕咕的教学视频
