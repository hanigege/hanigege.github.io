出国自建voip(三.2）Telegram机器人连接程序自启动（补充篇）

root@sms:~# source myenv/bin/activate

激活虚拟环境

source myenv/bin/activate 

(myenv) root@sms:~# nohup python sms3.py > sms.log 2>&1 &

后台启动sms3.py

nohup python sms3.py > sms.log 2>&1 &
or
nohup python sms.py > sms.log 2>&1 &
(myenv) root@sms:~#

上面的两条命令能不能做成脚本，设置为开机自启动

可以的，你可以创建一个 systemd 服务 或者 编写一个 shell 脚本 来实现 Python 虚拟环境的激活 并 运行 sms3.py 脚本，同时设置为 开机自启动。

方法 1：使用 systemd (推荐) 适用于 Debian/Ubuntu 及大多数 Linux 发行版。

步骤 1：创建 systemd 服务文件 运行以下命令，创建 sms3.service：

nano /etc/systemd/system/sms3.service

填入以下内容：

[Unit]
Description=SMS Python Service
After=network.target

[Service]
User=root
WorkingDirectory=/root
ExecStart=/root/myenv/bin/python /root/sms3.py
Restart=always
StandardOutput=append:/root/sms.log
StandardError=append:/root/sms.log

[Install]
WantedBy=multi-user.target

⚠ 注意：

WorkingDirectory=/root 需要修改成你的 Python 虚拟环境和 sms3.py 所在目录。

ExecStart=/root/myenv/bin/python /root/sms3.py 确保路径正确。

Restart=always 让进程崩溃后自动重启。

步骤 2：启用并启动服务

systemctl daemon-reload
systemctl enable sms3
systemctl start sms3
查看运行状态：

systemctl status sms3
如果 显示 active (running) 说明成功。
